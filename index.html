캡스톤 디자인 시작


규칙

1. 의견이 달라도 서로 상처받지 않기

2. 관련 없고 이상한 트집 잡으면 사형

3. 해야할 일 안 하고 빈둥거리면 능지처참


주제

인구밀집도가 높은 대한민국의 주차 시스템 개선


주제 선정 이유

1. 주차장 전광판에 주차가능 표시가 보여도 비어있는 자리를 찾기가 상당히 힘듦

2. 빈자리 찾았다고 싱글벙글 가면 꼭 경차가 있음



구현 목표

1. 주차장 바닥의 일부만 디스플레이로 만들어서화살표 모양의 가이드가 출력되게 할 수 있어야함.

2. 주차장 입구에 이미지 인식이 가능한 AI를 통해 방문 차량이 경차인지 중형차인지 구분할 수 있어야함.

3. 회원가입을 통해 미리 주차 자리를 예약할 수 있고 결제할 수 있는 웹페이지가 구현되어야 함

4. 웹이나 앱이 주차장과 데이터를 주고 받으며 게이트와 통신이 가능해야함



웹 사이트 구현 방법

1단계 : html과 css, javascript를 이용해서 페이지의 뼈대를 잡는다.

2단계 : Mysql의 DB를 이용해서 사용자가 입력한 정보를 관리하고 회원가입 서비스를 제작한다.

3단계 : 결제는 어떻게 구현하는건지 구글링 열씌미 해본다.

4단계 : 주차장과 통신하는 방법을 열씌미 구글링 해본다.

상시 : 에러가 발생하면 언제든지 1단계로 돌아가고 UI를 지속적으로 업그레이드하며 GIT-HUB에 과정을 꼭 기록한다
